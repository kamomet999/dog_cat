## 1. コンセプトの再定義（v1からのピボット）

### 1.1 一言でいうと

> **スマホを触らないほど、犬や猫が育って増える。**
> Forest の「木」を「いぬ・ねこ」に置き換えたデジタルデトックス×コレクションアプリ。

### 1.2 v1との違い

| 観点 | v1（ACTION_PLAN.md） | **v2（本書）** |
|---|---|---|
| コアループ | 通知で呼び戻し、世話をさせる（接触を増やす） | **スマホから離れさせる**（接触を減らす） |
| 成長の源泉 | 世話アクション | **触らなかった時間** |
| コレクション | おまけ | **図鑑が主目的**（埋める面白さ） |
| 形態 | PWA | **ネイティブ（Capacitor）→ App Store / Google Play** |
| 期間 | 4〜6週 | **12日（6/10〜6/22）** |

### 1.3 コアループ

```
「おさんぽ」開始（30分/1h/2h を選ぶ）→ アプリを閉じてスマホを置く
  → 時間満了後に戻ると：おやつ(コイン)獲得＋ペット成長＋新しい子と出会うチャンス
  → 途中でアプリに戻ってしまうと「おさんぽ失敗」（ペットがしょんぼり・報酬なし）
  → 育った子は図鑑に登録。次の卵へ → 図鑑コンプを目指す
```

- **セッション型（おさんぽ）** = Forest型の能動デトックス
- **常時放置成長** = アプリを閉じている間も緩やかに成長（既存エンジンを流用）。二層構造で「開かない方が得」を徹底する
- 世話アクション（ごはん等）は **帰宅後の儀式** として残す（1セッション1回ボーナス程度に弱める）

### 1.4 図鑑（コレクション）設計

- 犬・猫 各15種以上、**リリース時30種**（既存 `breeds.js` を拡張。プロシージャルSVGなので追加コストが低い）
- レアリティ5段階。長時間おさんぽ・連続成功でレア出現率UP
- 図鑑画面：シルエット表示→出会うと開放。達成率%表示
- 成長段階：卵→赤ちゃん→子→成体（既存実装）。成体まで育てると図鑑に「殿堂入り」

### 1.5 技術的に正直な制約（重要）

「他アプリを触ったか」のOS横断検知は、iOSのScreen Time API・AndroidのUsageStatsとも審査・権限ハードルが高い。MVPは Forest と同じ割り切り：

- **判定は「自アプリへの早期復帰」のみ**。途中でアプリを開いたら失敗。画面オフ・スマホを置いておくのはOK
- 厳密な「スマホ全体の不使用検知」は v2.0 以降の検討課題として明記

## 2. 技術選定（12日で2OSに出すための構成）

| レイヤ | 採用 | 理由 |
|---|---|---|
| アプリ本体 | **既存の素のJS Webアプリを継続** | 動くプロトが既にある。12日でフレームワーク移行は悪手 |
| ネイティブ化 | **Capacitor 6** | 既存WebアプリをほぼそのままiOS/Androidアプリ化。学習コスト最小 |
| 通知 | @capacitor/local-notifications | 「おさんぽ完了」「ペットが会いたがってる」。サーバー不要 |
| ライフサイクル | @capacitor/app（appStateChange） | おさんぽ失敗判定・オフライン進行の基点 |
| 保存 | @capacitor/preferences（localStorageから移行） | アプリ削除まで永続。既存セーブ構造を流用 |
| iOSビルド | Xcode（Mac）または **Codemagic / Ionic Appflow**（Macが無い場合のクラウドビルド） | 後述 |
| アート | プロシージャルSVG（既存 art.js）＋AI生成のアイコン/ストア素材 | 権利クリーン・追加品種が安い |
| バックエンド | **なし**（完全ローカル） | 期日厳守・プライバシー申告も最小で済む |

## 5. ストア公開までの過程（解説）

### 5.1 iOS（App Store）

1. **Apple Developer Program 登録** — $99/年。個人なら法人書類不要。承認まで通常24〜48h
2. **ビルド** — Capacitorが生成するXcodeプロジェクトをアーカイブ。**Macが必須**。無い場合は Codemagic / Ionic Appflow 等のクラウドMacでビルド可能（無料枠あり）
3. **署名** — Xcodeの「Automatically manage signing」でほぼ自動。AppleIDでログインするだけ
4. **App Store Connect** にアプリ登録 → ビルドをアップロード
5. **TestFlight** で内部テスト（審査不要・即配布。実機確認はここで）
6. **審査提出** — 掲載情報（説明文・スクショ6.7"と5.5"・プライバシーポリシーURL・プライバシー「栄養表示」）を入れて提出
7. **審査** — 通常24〜72h。本アプリは課金なし・通信なし・個人情報なしなのでリジェクトリスクは低い。よくある指摘は「スクショと実画面の不一致」「メタデータの誇大表現」
8. **公開** — 承認後、手動 or 自動でリリース

### 5.2 Android（Google Play）

1. **Google Play Console 登録** — $25 買い切り。本人確認（身分証）で数日かかることあり
2. **ビルド** — `bundleRelease` でAAB生成。署名は **Play App Signing** に任せる（アップロード鍵だけ自前管理）
3. **内部テスト** — 最大100人、即時配布。実機確認用
4. **⚠ 個人の新規アカウントの場合**: 製品版公開の前に **「12人以上のテスターで14日間のクローズドテスト」が必須**（2023年11月以降の新規個人アカウントに適用）
   - → **6/22までにGoogle Playで一般公開は構造的に不可能**。6/20にクローズドテストを開始すれば **7月上旬に製品版公開** が最短
   - テスター12人は友人・家族・テスター募集コミュニティ（/r/AndroidClosedTesting 等）で確保。**6/16までに招集開始**
   - 既存の（2023年以前の）アカウントや法人アカウントを使えるならこの制約はない
5. **データセーフティフォーム** — 本アプリは「データ収集なし」で申告（完全ローカル設計の利点）
6. **製品版公開** — テスト要件クリア後、申請→審査（数時間〜数日）→公開

### 5.3 共通で必要なもの（AIが下書き、人間が確認）

- プライバシーポリシーURL（GitHub Pagesに静的ページを設置）
- アプリアイコン(1024px)・フィーチャーグラフィック(Play用1024×500)・スクリーンショット各サイズ
- 年齢区分質問票（暴力・課金なし → 全年齢）
- アプリ名・サブタイトル・説明文（日本語＋英語）

## 6. リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| 開発者アカウント承認の遅延 | 提出不能 | **今日(6/10)申請**。これより後はすべて遅延に直結 |
| Macが無い | iOSビルド不能 | Codemagicクラウドビルドに即切替（D2に判明させる） |
| Google Play 14日テスト要件 | 6/22一般公開不可 | 期待値を「6/22=iOS公開＋Playテスト開始、Play製品版は7月上旬」に設定 |
| iOS審査リジェクト | 数日の手戻り | 通信なし・課金なし・収集なしの最小構成で提出。6/21をバッファに |
| 画面オフ＝失敗と誤判定 | コア体験の崩壊 | appStateChangeの挙動を**D6の実機テストで最優先検証**。NGなら「復帰時刻判定方式」へフォールバック |
| スコープ膨張 | 未完 | §4スコープ外リストを死守。追加案はすべてv1.1へ |
