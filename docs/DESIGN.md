## 1. ポジショニングから導くデザインの大方針

調査で確認できた市場の構図：

| 陣営 | 代表 | 訴求 | 状況 |
|---|---|---|---|
| 緑・自然・ミニマル | Forest / Flora / Focus Plant | 規律・集中・自然メタファー | 老舗が密集するレッドオーシャン |
| **暖色・キャラ・世話** | **Focus Friend / Finch** | 愛着・「この子のために」 | **近年の勝者**（全米1位・ARR$30M） |

Focus Friendの成功要因は「**罪悪感ではなく、かわいい存在への責任感**で行動が変わる」こと（レビュー言説）。Finchは「かわいさは装飾ではなく感情の設計」。現行デザイン（クリーム×オレンジ・丸ゴ・ひらがな）は**すでに勝ち筋側にいる**ため、転換ではなく徹底を行う。

### デザイン原則（4つ）

1. **この子のために置く** — UIは常にペットの感情を主語にする（「失敗しました」ではなく「しょんぼりしてる…」）
2. **ゆるくて、責めない** — 高彩度・赤・警告アイコン・数字の詰め込みを避ける。低彩度×余白×丸み
3. **生きてる感** — 静止画を置かない。呼吸・まばたき・反応（Finchの定石）
4. **やわらかさは色相で出す。明度差は削らない** — 「薄い=やさしい」は誤り。茶系・暖色でやわらかく、コントラストはAA基準を守る（おさんぽ=屋外直射日光下で使うアプリである）

### 変えない約束（収益化との境界）

将来の収益化が「広告ゼロ・無課金コンプ」の旗を折らないためのデザイン制約（MARKETING.md §3 と対応）：

1. **図鑑画面・おむかえ抽選・おさんぽ報酬に課金導線を置かない**（30種コンプは永久に無料・広告ゼロ）
2. 課金が入る場合は「着せ替え・応援」の**別導線に隔離**し、未購入でも機能差ゼロ
3. §7 の「引き算」原則は**デフォルト状態の見た目**に適用する（ユーザーが選んだ着せ替えはこの限りではない）
4. **課金による「いのち」の復活・延命は永久に置かない**（命の重さの一線。GAME_DESIGN.md §3）

## 2. カラーシステム

### ひる（メイン）パレット

実測コントラスト比（WCAG 2.1）に基づき、**役割を fill（塗り）/ text（文字）に分離**する。色に2役を兼ねさせない。

| トークン | 値 | 用途 | 検証値 |
|---|---|---|---|
| `--bg1/--bg2` | #fdf6ec / #f6e7d8 | 背景（クリーム→ベージュ） | — |
| `--accent/--accent-d` | #ff9a76 / #f47b54 | 主ボタン・強調の**塗り専用**（唯一の大面積高彩度） | — |
| `--on-accent` | #4a3f35 | **[凍結前]** アクセント地の上の文字。**白は禁止**（白on#ff9a76=**2.07**で不合格） | **4.94** |
| `--ink` | #4a3f35 | 本文（真っ黒を使わない） | on bg1 **9.53** |
| `--ink-soft` | **#6f6354** | **[凍結前]** 補助文字（旧 #8a7d6d は on bg1 **3.74** で不合格） | bg1 **5.45** / bg2 **4.83** / 白 **5.85** |
| `--coin` | #f2b134 | コインの**塗り専用**。文字色に使わない（on白=**1.89**） | — |
| `--coin-text` | **#9c6d10** | **[凍結前]** 報酬数字などコイン系の文字 | on白 **4.56** |
| `--good/--warn/--bad` | #7ac77a / #e8a13a / #e06464 | ステータスバーの塗りのみ。**UIの警告・文字には使わない** | — |

#### ルール（検証可能な形）
- **文字は通常文字AA（4.5:1）以上**。例外は実装前にこの表へ実測値付きで追記する
- **大面積の塗り（ボタン・背景）に使える高彩度はアクセントのみ**。バー・バッジ・チップ等の小面積セマンティック色は対象外
- 影は黒ではなく茶系（`rgba(120,90,60,…)`）／新色追加前に既存トークンで代替できないか確認する
- 色地バッジの文字: coin地は `--ink`（**5.41**）、`badge-new` は地色を `#c2453f` に **[凍結前]**（白文字 **4.98**。旧 #e06464 は 3.40）

### レアリティ色（breeds.js RARITY）

- **レアリティは色のみで伝えない。主符号は★の数、ラベル併記、色は従**（現実装の3重符号化を規定として固定。色覚多様性対応）
- 白文字チップが成立するよう暗色版へ差し替え **[凍結前]**（旧値はコモン2.64・レジェンド2.12等で不合格）:
  コモン `#6d747c`(**4.73**) / アンコモン `#2e8757`(**4.45**) / レア `#2f6fc4`(**5.01**) / エピック `#8a42c8`(**5.68**) / レジェンド `#a87613`(**3.99**・11px太字運用)

### よる（おさんぽ）パレット

おさんぽ中画面の `#2e4a3a → #1d3328` を「夜の散歩」として正式採用。本文 `#cfe3cf` は実測 **7.2〜10.0:1** — **よるの本文はこれ未満の明度にしない**。アプリ全体のダークモードは **[v1.1]**（Pokémon Sleepのダーク非対応批判に基づく優先度付け）。

## 3. タイポグラフィ

スケール: Display 20-22px/800（ペット名・見出し。タイマーのみ56px tabular-nums）／Title 14-18px/800（ボタン・節見出し）／Body 12-13px・行間1.6-1.7（説明文）／Caption 10-11px/700（ラベル・チップ、`--ink-soft`可）

- **最小フォントサイズは10px**。現状の8px（new-dot）・9px（count-dot・★）は10pxへ引き上げ **[凍結前]**
- **漢字かな交じりを基本にする**（発案者FB 2026-06: 全ひらがなは読みにくい）。**内容語（名詞・動詞）は漢字、助詞・送り仮名・ペットの感情ボイス（「ただいま！」等）はひらがな**でやわらかさを担保。ターゲットは「やさしい一般的な日本語」＝LINE風。全ひらがなには戻さない
- フォントスタック: `"Hiragino Maru Gothic ProN", "Hiragino Sans", "Yu Gothic", system-ui, sans-serif`
- **Androidに丸ゴは存在しない**（Noto Sans JP=角ゴに落ちる）。よって**丸み・ゆるさはフォントに依存させず、形状（radius 22px）・配色・ひらがなコピーで担保する**のが本則。角ゴ表示でも世界観が成立することを確認してからリリース。見出し用丸ゴサブセット同梱（数十KB以内）は **[v1.1]**

## 4. モーション

2層に分ける。**新しい duration / easing を発明しない**。

- **UI遷移**: `--dur-fast:100ms`（pressed）/ `--dur-base:200ms`（モーダル・トースト）/ `--dur-slow:400ms`（バー伸縮）。easing は ease-out 基本
- **環境（生きてる感）**: 2.4〜3.4s / ease-in-out / infinite 可（呼吸3.4s・おさんぽ浮遊2.4s）。**transform のみ使用**（layout/paint を起こすプロパティ禁止＝バッテリー配慮）
- 演出系（bounce/wiggle/pop）は0.45-0.5s・1回再生のみ（既存の reflow 再生方式を維持）
- **[凍結前]** `@media (prefers-reduced-motion: reduce)` で infinite 系（breathe / walkfloat）を停止。UI遷移は残してよい（前庭障害への配慮、数行で入る）

## 5. コンポーネント状態

- pressed: `:active` で scale .92-.96・`--dur-fast`（現行）
- disabled: `opacity .45-.5`＋`cursor:not-allowed`。**グレー差し替えはしない**（現行）
- focus: **[凍結前]** `:focus-visible { outline:2px solid var(--accent-d); outline-offset:2px }` を全ボタンに（現状0件）
- エラー（コイン不足等）: トーストで感情主語の一言＋disabled のみ。**赤・警告アイコンを出さない**（原則2）
- 空状態（図鑑0匹）: 肉球モチーフ＋「さいしょの子をそだてよう」 **[v1.1]**／ロード状態はローカル完結のため原則不要
- 図鑑グリッド: 3列・サムネ74px維持。未発見シルエットの `opacity` を **.22 → .32** に **[凍結前]**（現状は実効約1.6:1で屋外で見えない）。NEW/×n バッジは10px以上・色地ルール（§2）に従う
- **差別化の旗の可視化** **[凍結前]**: 図鑑の**フッター余白**に1行のみ「**ぜんぶの子と、むりょうで あえるよ。こうこくも、ないよ**」（`--ink-soft`・Caption 10-11px・ヒエラルキー最下位）。出す場所はアプリ内でこの1箇所に限定（原則2「責めない」を壊さない）。この画面の実写をストアスクショ5枚目に使う（§10）

## 6. 触覚・サウンド

- **v1.0 は意図的にサイレント**。効果音・BGMは入れない（「スマホから離れる」アプリが音で注意を引くのは自己矛盾。沈黙は機能である）

## 7. キャラクター規定（ゆるかわ文法）

調査で複数ソース確認できた「ゆるかわ」構成要素と、本アプリでの実装状態：

| 要素 | 根拠 | 実装状態 |
|---|---|---|
| 頭身約2:1・丸いフォルム | ベビースキーマ（保護本能の自動刺激） | ✅ 現行: 頭r48 vs 体r42〜50 |
| つぶらな目＋ハイライト | 同上 | ✅ 現行 |
| ほっぺの赤み（常時） | 同上 | ✅ 現行（通常0.35-0.38・喜び0.55-0.6・しょんぼり時は消える） |
| アイドル時の「生きてる感」 | Finchの emotional engineering | ✅ 現行（3.4秒周期のゆっくり呼吸。reduced-motion時は停止 **[凍結前]**） |
| なでると反応 | 同上 | ✅ 現行（bounce/wiggle） |
| 手描きの線の揺らぎ | ゆるかわ文法（ラフ線が緊張を解く） | ⏩ **[v1.1]** プロシージャルSVGに輪郭ゆらぎを足す実験 |
| 呼べる名前（ユーザー命名） | 愛着・UGCの核（「うちの◯◯が」と語れる。Finch/たまごっちの定石）。原則1を最も強く実装する要素 | ⏩ **[v1.1]** めざめ時に命名ダイアログ（スキップ可・デフォルトは品種名） |
| 性格（30種それぞれに性格タイプ＋一言） | 「うちの子は◯◯な性格」という語り＝愛着とUGCの核 | ✅ 現行（15タイプ。ホームのチップ＋図鑑詳細に一言表示） |
| 世界観の生物学的整合（哺乳類に卵を使わない） | 没入感・レビュー指摘の予防 | ✅ 現行（stage0は「おくるみ」= ねんね中の赤ちゃんをおむかえ→めがあいて品種判明） |
| おかえり反応（成功おさんぽ後の初回起動でペットが駆け寄る/はねる1回演出） | 復帰時=セッション最大の感情ピーク（Finchの定石） | ⏩ **[v1.1]** 既存 bounce の流用で軽量 |
| 要素の引き算 | ねこあつめ「複雑さを入れない」 | ✅ 維持。装飾追加の誘惑に抵抗する（デフォルト状態に適用。§1「変えない約束」参照） |

SVG実装の禁則: clipPath/グラデの id を使わない（多数描画時のid衝突防止）／模様は本体内座標に収める／影は茶系。

## 8. UIコピー・通知（ボイス&トーン）

本節の管轄は **UI・プッシュ通知・シェア文言・ストア文言のすべて**（ブランドが言葉を発するところ全部）。

- ひらがな優先・漢字は最小限（現行どおり）
- ペットの感情を主語に: ❌「セッション失敗」→ ⭕「ちょっとしょんぼりしてる…」
- 命令しない: ❌「スマホを置いてください」→ ⭕「スマホをふせて、まっててね」
- 失敗時も次を約束: 「つぎはきっとだいじょうぶ！」（実装済み）
- **効能を約束しない**: ❌「スマホ依存がなおる」→ ⭕「スマホとちょっと距離を置く」。医療・健康効果の断定表現はストア文言含め禁止（審査リスク兼ブランド規定）

### 通知の規定

- 通知もペットの感情を主語にする（現行 `native.js`「むかえにきてね。ごほうびがまってるよ」を正とする）
- **通知は「良い知らせ」＋「いのちに関わるお願い」のみ**（2026-06-12 生存システム導入に伴い改訂。GAME_DESIGN.md §4）。危険予告はペットのお願いとして書く（「おなかぺこぺこだよ…」）。罪悪感・催促・再エンゲージ目的の通知は引き続き禁止。**おさんぽ失敗時の通知も出さない**
- 通知の絵文字は1個まで（🐾 を標準とする）

### 固有名詞の表記規定

- アプリ名表記は「**いぬねこ図鑑**」で固定（「いぬねこずかん」「イヌネコ図鑑」を公式には使わない。SNS検索回収のため）
- 公式ハッシュタグは「**#いぬねこ図鑑**」1本。シェア文言・ストア説明文末尾・告知投稿で同一表記
- シェア文言テンプレの正: 「◯時間スマホをはなれて、◯◯（品種名）がそだちました #いぬねこ図鑑」（品種名という固有性の高い語が自然に入る構造）

## 9. アプリアイコン

- **採用: 柴犬の顔アップ（笑顔・ほっぺ）**。調査結果「カジュアル・コンシューマー系では『フレンドリーなキャラ顔』の勝率が高い」「最小サイズでの判読性・支配的要素1つ」に合致
- 肉球マークは**ブランドモチーフ**としてスプラッシュ・空状態・装飾に継続使用（`assets/icon-paw-variant.png` に保存）
- 公開後、ストアのA/Bテスト（Play実験/PPO）で 顔 vs 肉球 を検証する（アイコン変更は2桁%のCVR差が出る変数）
- **[ストア素材]** A/Bの第1ラウンドは「柴犬の顔」の**表情・背景色バリエーション**（例: 背景クリーム vs アクセント橙）から行う。顔 vs 肉球 は第2ラウンド以降。**1回のテストで動かす変数は1つ**

## 10. ストアスクリーンショットの公式（調査ベース）

- **最初の3枚が約70%のコンバージョン重量・閲覧7秒** → 1枚目に機能説明を置かない
- 構成は **Value → Usage → Trust**:
  1. 「スマホを置くと、育つ。」— **ペットの顔が最大要素（画面の1/3以上）**。UIの情報密度より表情を優先し、検索結果サムネ（縮小時）で目とほっぺが判別できること（§9のアイコン採用根拠と同じ論理）
  2. 「30分の おさんぽから」（おさんぽ画面）
  3. 「ぜんぶ無料で 30種コンプ」（図鑑）
  4. 使い方3コマ / 5. 「広告ゼロ・登録不要」— **図鑑画面の実写**を使う（§5の旗の1行が写ること。「ストアの宣伝文句」ではなく「アプリ内の事実」として見せる）
- コピーは**1行・3〜5語・サムネイルで読める太さ**（拡大して見る人は4%しかいない）
- iOS=世界観ポスター寄り / Google Play=明るい背景＋実UI寄りに作り分け
- **Google Play フィーチャーグラフィック**（掲載面最上部の横長画像・スクショより先に見られる）: ペットの顔＋「スマホを置くと、育つ。」の1メッセージのみ。サムネ判読性ルールを適用 **[ストア素材]**
- **アプリプレビュー動画は v1 では作らない**（静止画で世界観が完結する型・制作コスト対効果が低い）。検討するなら「呼吸する待機アニメ15秒ループ」のみを素材化する v1.1 以降の議題とする
- ※ `docs/store/listing.md` のスクショ構成は本節の並びを正とする（2026-06-10 改稿済み）

## 10.5 シェアされる瞬間のデザイン（獲得装置）

獲得チャネルがシェア・口コミのみである以上、**ユーザーが撮るスクショ1枚1枚が唯一の広告枠**。共有API実装は凍結に間に合わないが、「自力スクショが広告として機能するか」はモーダルのCSS/文言で決まる。

- **[凍結前]** 成功モーダルを「自慢の1枚」フォーマットに: ①ペット絵（大） ②品種名 ③「◯時間 スマホをはなれた」 ④れんぞく回数 ⑤画面下部に小さくウォーターマーク（「いぬねこ図鑑 🐾」1行・10px・`--ink-soft`・主張しない）を**1画面に収める**。CSS＋文言の組み替えのみ（現行 `ui.js` showWalkSuccess には品種名・アプリ名が無い）
- **[凍結前]** 図鑑コンプ系の節目モーダル（10種・20種・30種）にも同じウォーターマーク規定を適用
- **[v1.1]** Capacitor Share API による共有カード自動生成。世界観規定: ひるパレット背景・ペット絵中央・コピーは§8準拠・ウォーターマーク必須。**ハッシュタグは画像に焼き込まず、Share の共有テキスト側に入れる**（§8のテンプレ使用。宣伝要素が2つ写った画像は「広告っぽさ」が出てシェア意欲を下げる＝引き算原則）。命名機能（§7）実装後は「名前＋品種名」併記を正とする
