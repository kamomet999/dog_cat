# デザイン方針書レビュー — 01 / シニアプロダクトデザイナー

- レビュー日: 2026-06-10（コード凍結 6/22 の12日前）
- 対象: `docs/DESIGN.md`／実装照合: `www/css/styles.css`, `www/js/art.js`, `www/js/ui.js`, `www/js/breeds.js`
- 総評: **戦略の上半分（ポジショニング・ゆるかわ文法・コピー）は良い。下半分（実装を縛る規定）がほぼ無い。**
  現状の方針書は「なぜこの世界観か」には答えるが、「このボタンの文字色は何か」「アニメは何msか」に答えられない。
  AIエージェントが実装者である以上、検証可能なルールが無い項目は毎回その場の判断で書かれ、ばらつく。
  以下、コントラスト比は WCAG 2.1 相対輝度式で実測した。

## 重要度: Critical（凍結前に必ず直す。全てCSS/定数の1行級修正）

### C-1. 主ボタン: 白文字 × アクセント橙が 2.07:1（AA 4.5:1 の半分以下）
- 実測: `#fff` on `#ff9a76` = **2.07:1**、グラデ終端 `#f47b54` でも **2.69:1**。ラベルは15px/800なのでWCAGの「大きい文字」(18.66px bold〜)にも該当せず、AA(4.5:1)・大文字AA(3:1)とも不合格
- アプリの最重要CTA（おさんぽ・おむかえ）が一番読めない、は致命的。屋外（=おさんぽ文脈）の直射日光下では実害が出る
- 修正案: ボタン文字を `--ink #4a3f35` に変更 → **4.94:1（AA合格）**。暖色地×焦げ茶文字はパレットの世界観とも整合する。`--on-accent` トークンとして定義
- 該当: `styles.css` `.big-btn.primary { color:#fff }`

### C-2. コイン色 #f2b134 を「文字色」に使っている — 1.89:1
- 実測: `#f2b134` の文字 on 白カード = **1.89:1**。`.care-btn .cost`(10px!)、報酬表示 `color:var(--coin)` が該当。報酬という「一番気持ちいい瞬間」の数字がほぼ読めない
- `count-dot`（白文字 on coin地）も **1.89:1**
- 修正案: テキスト用トークン `--coin-text: #9c6d10` を追加（on 白 = **4.56:1**）。`--coin` は塗り（コインの円・バッジ地）専用とし、coin地の上は `--ink`（**5.41:1**）。「色には text/fill の2役を兼ねさせない」をルール化

### C-3. レアリティチップ: 白文字が5色中3色で不合格
- 実測（白文字 on 色地）: コモン `#9aa0a6` **2.64**・アンコモン `#3fa86b` **2.99**・レジェンド `#e0a93a` **2.12**（不合格）／レア `#3f7fd6` **4.02**・エピック `#9a4fd6` **4.70**（合格）
- 一番レアな「レジェンド」が一番読めないのは演出として逆効果
- 修正案: `breeds.js` の RARITY 色を暗色版に差し替え（コモン `#6d747c`=**4.73**、アンコモン `#2e8757`=**4.45**、レア `#2f6fc4`=**5.01**、エピック `#8a42c8`=**5.68**、レジェンド `#a87613`=**3.99**※11px boldなら太字運用で許容、厳密にAAなら `#9c6d10`）。チップ・★・図鑑詳細すべて同一定数を参照しているので1箇所の変更で済む

### C-4. prefers-reduced-motion 対応が無い（呼吸・浮遊が無限ループ）
- `breathe`(3.4s infinite)・`walkfloat`(2.4s infinite) は前庭障害ユーザーに有害になり得る区分の動き。OS設定「視差効果を減らす」を無視している。常時コンポジットはバッテリーにも効く（デトックスアプリが電池を食うのはブランド毀損）
- 修正案: `@media (prefers-reduced-motion: reduce)` で infinite 系を停止、遷移は残してよい（数行のCSSで凍結前に入る）

## 重要度: High（凍結前に入れたい。軽量）

### H-1. ink-soft #8a7d6d が補助文字として不合格
- 実測: on `--bg1` = **3.74**、on `--bg2` = **3.31**、on 白カード = **4.01**。用途は11-12pxの補助文字（成長ラベル・図鑑名・説明）で、小サイズ×低コントラストの二重苦
- 修正案: トークン値を `#6f6354` に変更 → bg1 **5.45** / bg2 **4.83** / 白 **5.85** / 図鑑セル地 **5.35**。見た目の「やわらかさ」は保ったままAA合格。1行修正
- 「真っ黒を使わない」原則は正しいが、「薄ければやわらかい」は誤り。**やわらかさは色相（茶系）で出し、明度差は確保する**を原則化すべき

### H-2. タイポグラフィの規定が丸ごと欠落
- 方針書にフォント・サイズ・行間の規定が一切無い。実装は8px（new-dot）/9px（count-dot・★）/10px/11px…とアドホックに散っている。8-9pxはどの端末でも判読限界以下
- **Hiragino丸ゴはiOS/macOS専用**。現スタック `"Hiragino Maru Gothic ProN","Hiragino Sans","Yu Gothic",system-ui` はAndroidでは全部すり抜けて Noto Sans JP（角ゴ）に落ちる。つまり**Androidユーザーは「丸ゴのゆるかわ」を一度も見ない**のに、方針書はこれに言及していない
- 修正案（凍結前）: 最小フォント10pxの床を敷く（8/9px箇所を引き上げ）。タイプスケールを規定（下記改稿に記載）。Androidは「角ゴ前提でも成立するよう、丸みはフォントでなく形状・配色・コピーで担保する」と明文化
- 修正案（v1.1）: M PLUS Rounded 1c 等のサブセット同梱（見出しのみ・約100字なら数十KB）を検討

### H-3. モーションのトークンが無い
- duration が .1/.18/.2/.25/.4/.45/.5/2.4/3.4s と9種、easing もバラバラ。新しいUIをエージェントが書くたびに新しい値が生まれる構造
- 修正案: `--dur-fast:100ms / --dur-base:200ms / --dur-slow:400ms / ease-out基本` をトークン化し、「環境（呼吸・浮遊）系のみ2.4-3.4s・ease-in-out・infinite可」と2層に整理。既存値はほぼこの近傍なので、規定を敷くだけで凍結前コスト極小

### H-4. フォーカス・状態規定の欠落
- `:focus-visible` スタイルが0件。キーボード・スイッチコントロール・TalkBackで現在位置が見えない。`outline` をアクセント色2pxで全ボタンに（3行）
- disabled は `opacity:.45/.5` で実装済みだが方針書に無い。pressed（`:active` scale .92-.96）も実装はあるが規定が無い → 改稿で「状態マトリクス」として明文化し、エラー状態（コイン不足等）は「トースト＋ボタンは無効化のみ、赤色は使わない」をルール化（現実装と一致、追コスト0）

### H-5. 図鑑シルエットが薄すぎる（opacity .22）
- 未発見セル: `brightness(0) opacity(.22)` on `#faf4ea` ≒ 実効コントラスト約1.6:1。「いる気配」の演出意図は良いが、屋外では存在自体が見えない。収集欲を起こす装置なのに視認できないのは本末転倒
- 修正案: `.32〜.35` へ（演出は保ちつつ輪郭が判る）。1行修正

## 重要度: Medium

### M-1. レアリティの色覚多様性 — 実装は偶然合格、規定が無い
- 実装は色＋★数＋ラベルの3重符号化になっており、色だけに頼っていない（これは良い）。ただし方針書に規定が無いので、今後「色だけのソート」「色だけの枠線」が入るリスク → 「レアリティを色のみで伝えない（★数が主・色は従）」を明文化
- なおエピック紫/レア青はP型・D型で近接するが、★が常時併記なら許容

### M-2. 触覚・サウンドの方針が無い
- 実装には `Native.buzz()`（おさんぽ成功のみ）が既にあるが、方針書はゼロ言及。なで/世話/成長で鳴らすのか、音は入れるのか、エージェントが判断できない
- 修正案: v1.0は「触覚=成功・成長の2イベントのみ、軽く1回。音は無し（サイレント設計はデトックス文脈の機能である）」と決め打ちで明文化。コスト0（現状維持を規定化するだけ）

### M-3. 「高彩度色は同一画面に1つ」ルールが自己矛盾
- ホーム画面にはアクセント橙＋ステータス3色＋コイン金＋レアリティ色が同居しており、ルールとして検証不能=形骸化している
- 修正案: 「**面積の大きい塗り（ボタン・背景）に使える高彩度はアクセントのみ。バッジ・バー・チップ等の小面積セマンティック色は除外**」と検証可能な形に書き直す

### M-4. ロード/空状態の規定が無い
- ローカル完結アプリなのでロードはほぼ無いが、「初回（図鑑0匹）」の図鑑画面はシルエット30個が並ぶだけ。空状態に肉球モチーフ＋「さいしょの子をそだてよう」の一言を出す規定を（v1.1で可）

### M-5. 文書構造: 「今回実装した変更」(§7)は方針書に不要
- §7は changelog であって方針ではない。git log に任せ、方針書は「常に現在形の判断基準」だけにする。また各規定に「凍結前/v1.1」のタグが無く、エージェントが着手可否を判断できない

## 重要度: Low
- L-1. `badge-new` 白文字 on `#e06464` = **3.40**（8-10px）。`#c2453f` なら **4.98**。ついで修正可
- L-2. おさんぽ画面は実測 **7.2〜10.0:1** で優秀。「よるパレットの本文は #cfe3cf 以上」と良い実態を規定として固定しておくとよい
- L-3. 影の茶系統一・id衝突しないSVG設計はコメント含め良実装。方針書に「やってはいけないこと」として吸い上げる価値あり

## 良かった点（保持すべき）
- 市場ポジショニングからデザイン原則への接続（§1）は根拠が明確で説得力がある
- ゆるかわ文法の表（根拠×実装状態）は検証可能な書き方の手本。他セクションもこの形式に寄せるべき
- コピー原則（感情主語・命令しない・次を約束）は実装と完全に一致しており機能している
