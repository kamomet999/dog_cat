## 総評

デザイン方針書としての完成度は高い。§9（アイコン）・§10（スクショ公式）が方針書に入っている時点で、個人開発のデザインドキュメントとしては上位1%に入る。**問題は「獲得チャネル＝シェア・口コミのみ」という戦略に対して、"シェアされる瞬間"のデザイン規定がほぼ空白なこと**。広告費ゼロの戦略では、ユーザーが撮るスクショ・共有カードの1枚1枚が唯一の広告枠であり、これは「あれば良い演出」ではなく**獲得装置そのもの**。現状その規定は §12 バックログの1行（「シェア画像テンプレート」）しかない。

また、MARKETING.md が「シェア画像生成は【v1.0でも軽く】」と言っているのに、DESIGN.md では **[v1.1]** に落ちている。**この2文書の矛盾は凍結前に決着させるべき最重要論点**（→ 提案1）。

以下、提案ごとに【優先度】【区分】【根拠】【修正・追記案】の形式で記す。

## A. シェア・口コミ（広告費ゼロ戦略との整合）

### 提案1. 「おさんぽ成功モーダル」を"スクショされても広告になる"設計に格上げする
- 【優先度: 高】【区分: 凍結前（軽量版）＋ v1.1（完全版）】
- 【根拠】
  - MARKETING.md §4 は「シェア画像の自動生成」を **【v1.0でも軽く】** と明記。DESIGN.md は §12 で **[v1.1]** に後退させており、戦略文書と矛盾している
  - 共有API実装（Capacitor Share＋canvas）が凍結に間に合わないのは妥当な判断。だが**「ユーザーが自力でスクショを撮ったとき、その画像が広告として機能するか」は今のモーダルデザインだけで決まり、CSS/文言レベルで凍結前に直せる**
  - 現状の成功モーダル（`ui.js` L473前後）は「れんぞく成功 X 回め」等の数字はあるが、**アプリ名・品種名・デトックス時間が1画面に揃ったフレームになっていない**。スクショが流通しても「何のアプリか」が伝わらない
- 【修正・追記案】
  - DESIGN.md に **新節「§10.5 シェアされる瞬間のデザイン（獲得装置）」** を追加し、以下を規定:
    1. **[凍結前]** 成功モーダルは「自慢の1枚」フォーマットに: ①ペット絵（大）②品種名 ③「◯時間 スマホをはなれた」④れんぞく回数 ⑤画面下部に小さく「いぬねこ図鑑」のロゴテキスト＋肉球マーク（ウォーターマーク。`--ink-soft`・10px・主張しない）を**1画面に収める**。CSS＋文言の組み替えのみ
    2. **[凍結前]** 図鑑コンプ系の節目モーダル（10種・20種・30種）にも同じウォーターマーク規定を適用
    3. **[v1.1]** Capacitor Share APIによる共有カード自動生成（§12の既存項目）。カードの世界観規定: ひるパレット背景・ペット絵中央・コピーは§8のボイス&トーン準拠・ウォーターマーク必須・**推奨ハッシュタグ「#いぬねこ図鑑」をカード内に焼き込む**（提案3参照）
  - §12 の「シェア画像テンプレート」の行に「※凍結前にモーダル側の"スクショ映え"対応を先行する（§10.5）」と注記

### 提案2. 「広告ゼロ・無課金で30種コンプ」の旗をアプリ内のどこかに可視化する
- 【優先度: 高】【区分: 凍結前（文言1行）＋ ストア素材】
- 【根拠】
  - MARKETING.md の勝ち筋1は「広告ゼロ・無課金コンプを**公約にする**」。現状この旗が立っているのはストア説明文だけで、**アプリ内には一度も出てこない**。レビューを書く・友人に勧める瞬間にユーザーが思い出すのはアプリ内の体験であり、旗はアプリ内に立っていないと口コミの言葉にならない（スマやめユーザーの乗り換え動機を毎回想起させる効果もある）
  - 押し付けがましく出すと世界観（§1原則2「ゆるくて、責めない」）を壊すので、出す場所は1箇所に絞るべき
- 【修正・追記案】
  - DESIGN.md §5 の図鑑グリッド規定に追記 **[凍結前]**: 図鑑ヘッダー（または図鑑のフッター余白）に1行、`--ink-soft`・Caption サイズで「**ぜんぶのこと、むりょうで あえるよ（こうこくも ないよ）**」のような世界観準拠の固定文言を置く。視覚的ヒエラルキーは最下位でよい。文言は§8のトーン（ひらがな・感情主語寄り）に従う
  - §10（スクショ公式）に追記: 5枚目「広告ゼロ・登録不要」のスクショは**この図鑑画面の実写**を使う（旗が「ストアの宣伝文句」ではなく「アプリ内の事実」として見える。Trust枠の説得力が一段上がる）

### 提案3. 固有名詞・ハッシュタグの公式表記を方針書で1つに固定する
- 【優先度: 中】【区分: 凍結前（方針書追記のみ・コード変更なし）】
- 【根拠】
  - 「いぬねこ図鑑」は一般名詞の組み合わせで、X検索すると実在の犬猫図鑑・ペット系投稿に埋もれる。SNS上で言及を回収（エゴサーチ・UGC再シェア）するには、**表記ゆれのない公式タグと、検索で衝突しない固有の言い回し**が必要
  - 「おさんぽ」「デトックス」も一般語。アプリ固有の語彙が現状ゼロで、口コミが「スマホ置くと犬育つやつ」としか言えない＝検索でアプリに辿り着けない
- 【修正・追記案】
  - DESIGN.md §8（UIコピー）に「**固有名詞の表記規定**」小節を追加:
    - アプリ名表記は「いぬねこ図鑑」で固定（「いぬねこずかん」「イヌネコ図鑑」を公式には使わない。検索回収のため）
    - 公式ハッシュタグは「#いぬねこ図鑑」1本。シェアカード（提案1）・ストア説明文末尾・告知投稿で必ず同一表記を使う
    - 将来のシェア文言テンプレは「◯時間スマホをはなれて、◯◯（品種名）がそだちました #いぬねこ図鑑」を正とする（品種名という固有性の高い語が自然に入る構造。「コーギーが育った」は「魚が育った」より言及されやすい）

### 提案4. ペットに「呼べる名前」を付けられるようにする（ブランドの記憶可能性）
- 【優先度: 中】【区分: v1.1】
- 【根拠】
  - 実装確認: 現状ペット名は品種名がそのまま表示される（`ui.js` L119: `petName.textContent = stage === 0 ? 'たまご' : breed.name`）。**ユーザー命名機能がない**
  - Finch・Focus Friend・たまごっち系の口コミ核は「うちの◯◯が」という固有名の語り。命名は愛着（リテンション）とUGC（「うちのモチ太が巣立った」）の両方に効く、コスト対効果最大級の機能
  - §1原則1「この子のために置く」を最も強く実装するのは色でもアニメでもなく**名前**である
- 【修正・追記案】
  - DESIGN.md §7（キャラクター規定）の表に行を追加: 「呼べる名前（ユーザー命名） | 愛着・UGCの核（Finch/たまごっちの定石） | ⏩ **[v1.1]** 孵化時に命名ダイアログ（スキップ可・デフォルトは品種名）」
  - §12 バックログにも同項目を追加。シェアカード（提案1のv1.1版）は命名実装後「名前＋品種名」併記を正とする

## B. 獲得（ストア転換）

### 提案5. スクショ公式の「1枚目」に感情フックの規定を足し、listing.md との不整合を解消する
- 【優先度: 高】【区分: ストア素材（規定追記は凍結前でも可・コード外）】
- 【根拠】
  - §10の構成（Value→Usage→Trust・1枚目に機能説明を置かない）は正しい。ただし1枚目の規定が「ホーム画面＋大コピー」止まりで、**「顔」の規定がない**。カジュアル系の検索結果サムネで止まる要素はコピーではなくキャラの表情（§9でアイコンに「笑顔・ほっぺ」を規定したのと同じ論理が、スクショ1枚目にだけ適用されていない）
  - **DESIGN.md §10 と listing.md のスクショ構成が食い違っている**: DESIGN は3枚目に「ぜんぶ無料で30種コンプ」（差別化の旗をコンバージョン重量70%圏内に置く）、listing は3枚目「成功モーダル」・無料訴求はスクショに無し。差別化の旗が7秒圏外に落ちており、戦略上は DESIGN 側が正しい
- 【修正・追記案】
  - DESIGN.md §10 の1枚目規定を強化: 「1枚目は**ペットの顔が最大要素**（画面の1/3以上）＋コピー『スマホを置くと、育つ。』。UIの情報密度より表情を優先し、検索結果サムネ（縮小時）で目とほっぺが判別できること」
  - listing.md のスクショ構成案を DESIGN §10 の並びに合わせて改稿（1. 感情フック / 2. おさんぽ / 3. **無料コンプ訴求（図鑑）** / 4. 使い方 / 5. 広告ゼロ・登録不要）。※listing.md の修正は次のデザイナー or マーケ作業として起票
  - §10 に追記: 「**Google Play フィーチャーグラフィック**（検索結果・掲載面の最上部に出る横長画像）の規定 — ペット顔＋『スマホを置くと、育つ。』の1メッセージ。スクショより先に見られる枠であり、サムネ判読性ルールを適用」（現状この最重要素材への言及がゼロ）

### 提案6. アプリプレビュー動画は「v1では作らない」と明記する（意思決定の固定）
- 【優先度: 低】【区分: ストア素材（方針追記のみ）】
- 【根拠】
  - 動画プレビューはCVRを上げる場合と下げる場合が拮抗する素材で、制作コストが高い。本アプリは「静的スクショで世界観が伝わる」型であり、個人開発×6/22凍結の体制では作らない判断が正しい。ただし方針書に書いていないと、後から「動画も作るべきか」で迷走する
- 【修正・追記案】
  - DESIGN.md §10 末尾に1行: 「アプリプレビュー動画は v1 では作らない（静止画で世界観が完結する型・制作コスト対効果が低い）。検討するなら『呼吸する待機アニメ15秒ループ』のみを素材化する v1.1 以降の議題とする」

### 提案7. アイコンA/Bの「比較軸」を方針書の規定に揃える
- 【優先度: 低】【区分: ストア素材】
- 【根拠】
  - §9 の「公開後に顔 vs 肉球をA/B」は正しいが、肉球案は「最小サイズでのフレンドリーなキャラ顔の勝率」という自らの採用根拠と矛盾する対抗馬。負け筋の比較にスロットを使うリスクがある
- 【修正・追記案】
  - §9 に追記: 「A/Bの第1ラウンドは『柴犬の顔』の**表情・背景色バリエーション**（例: 背景クリーム vs アクセント橙）から行う。顔 vs 肉球は第2ラウンド以降。1回のテストで動かす変数は1つ」

### 提案8. プッシュ通知の文言を §8（ボイス&トーン）の管轄に入れる
- 【優先度: 高】【区分: 凍結前（方針書追記＋文言確認のみ・実装済み文言は概ね合格）】
- 【根拠】

  - 通知はユーザーがアプリ外で接触する唯一のブランド面＝リテンションの生命線。今後リマインド通知等を足すとき、規定がないとトーンが割れる（デトックスアプリの通知がうるさい、は最悪のレビュー導線）

  - §8 の題を「UIコピー・通知（ボイス&トーン）」に変更し、以下を追記:
    - 「通知もペットの感情を主語にする（現行『むかえにきてね。ごほうびがまってるよ』を正とする）」
    - 「**通知は『良い知らせ』のみ。罪悪感・催促・再エンゲージ目的の通知は出さない**（『最近来てないね』系はデトックスアプリでは自己矛盾でありブランド毀損）。失敗時の通知も出さない」
    - 「通知に絵文字は1個まで（🐾を標準とする）」

## D. 収益化との整合・リスク

### 提案10. 「無料の線引き」をデザイン原則として固定する（将来の収益化が旗を折らないように）
- 【優先度: 中】【区分: 凍結前（方針書追記のみ）】
- 【根拠】
  - MARKETING.md §3: 「収益化する日が来ても『図鑑コンプは無料』の線は守る（着せ替え・応援課金側で取る）」。これは経営判断ではなく**デザイン制約**（どこに課金導線UIを置いてよいかの規定）なのに、DESIGN.md に存在しない。将来の自分（またはAIエージェント）が図鑑内に課金バッジを置いた瞬間、最大の差別化が死ぬ
  - 一方で §7「装飾追加の誘惑に抵抗する」を硬く読むと着せ替え課金（v2想定）と衝突する。§6「v1.0は意図的にサイレント」も同様に「v1.0は」と期間が切ってあるのは良いが、§7には期間の限定がない
- 【修正・追記案】
  - §1 のデザイン原則の直後に「**変えない約束（収益化との境界）**」として3行追加:
    1. 図鑑画面・孵化抽選・おさんぽ報酬に課金導線を置かない（30種コンプは永久に無料・広告ゼロ）
    2. 課金が入る場合は「着せ替え・応援」の別導線に隔離し、未購入でも機能差ゼロ
    3. §7の「引き算」原則は**デフォルト状態の見た目**に適用する（ユーザーが選んだ着せ替えはこの限りではない）

### 提案11. ストア文言の過大表現を3点修正する
- 【優先度: 高】【区分: ストア素材（提出前必須）】
- 【根拠】（listing.md を実装と突合）
  1. **「30種いじょう」は事実と不一致**: `breeds.js` の品種数は**ちょうど30**。「以上」は審査上は通っても、数えるユーザー（図鑑アプリのコアユーザーは必ず数える）に「29種しかない」級の不信を生む種。日本のレビュー文化では誇張への減点が大きい
  2. **「完全無料」は将来の自分を縛る**: v2で着せ替え課金を入れた瞬間、説明文と「課金なし」の年齢区分回答の両方を書き換える羽目になり、既存ユーザーに「無料って言ったのに」と言われる。旗は「広告ゼロ」「図鑑コンプまで無料」であり「完全無料」ではない
  3. **デトックスの効能を健康効果として謳わない**: 現状の listing は概ね安全だが、規定がないと将来「スマホ依存が治る」系の文言が混入し得る（ヘルスケアカテゴリ選択時は特に審査リスク）
- 【修正・追記案】
  - listing.md: 「30種いじょう」→「**ぜんぶで30種**」（断言の方がコレクション欲を立てる。英語も "30+ breeds" → "all 30 breeds"）。品種追加のv1.1で「30種→35種に増えました」が逆にニュースになる
  - listing.md: 「完全無料・広告なし・登録不要」→「**広告ゼロ・登録不要。図鑑コンプまで課金はいっさい不要**」
  - DESIGN.md §8 に1行追記: 「効能を約束しない: ❌『スマホ依存がなおる』 ⭕『スマホとちょっと距離を置く』（医療・健康効果の断定表現はストア文言含め禁止）」

### 提案12. アプリ名のASO上の弱点を「検索順位ウォッチ項目」として固定する
- 【優先度: 低】【区分: ストア素材（リリース後運用）】
- 【根拠】
  - 「いぬねこ図鑑」単体は一般名詞で、本物の犬種・猫種図鑑アプリと検索結果を奪い合う（しかも相手はド本命の検索意図）。MARKETING.md §4-3 で認識済みだが、リネームは資産を捨てるので不要。**サブタイトル・キーワード欄で勝負する設計は listing.md で対応済み**であり、残るは運用の固定化のみ
- 【修正・追記案】
  - listing.md 末尾に「リリース後ASOウォッチ」小節を追加: 監視キーワード（スマホ依存 / デジタルデトックス / スマホ やめる / 集中 アプリ / 犬 育成 / 猫 育成 / ねこあつめ 系）と、「『いぬねこ図鑑』完全一致で自アプリが1位に出ない場合はサブタイトル変更を検討」のトリガー条件を書いておく

- §9・§10 がデザイン方針書に内包されている構造そのもの（アイコン・スクショは「デザインの一部」という認識は正しい）
