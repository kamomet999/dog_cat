# ブリード機能 仕様書（v1.1の目玉機能）

- 作成日: 2026-06-10 ／ 対象バージョン: **v1.1**（6/22のv1.0凍結後、Playクローズドテスト期間中に開発）
- ご要望: 「友人とペットをブリードして新しい子を産む。その際ランダムに引き継げる要素が欲しい」
- 設計方針: **サーバーなし・通信なしのまま実現する**（プライバシー「データ収集なし」申告を守る）

---

## 1. コンセプト — 「おみあい」

> 友だちの成体ペットと「おみあい」して、ふたりの特徴をランダムに受け継いだ赤ちゃん（おくるみ）をおむかえする。

- 機能名はUI上「**おみあい**」（世界観準拠・ひらがな。「ブリード」は開発用語）
- 哺乳類の世界観に整合：おみあい→おくるみ（ねんね中の赤ちゃん）が届く→めがあいて、どんな子か判明
- **同種のみ**（いぬ×いぬ、ねこ×ねこ）。犬×猫は不可＝生物学的整合性（「たまご」修正と同じ思想）

## 2. 通信なしでの実現 — おみあいコード

サーバーを立てずに、**コードのコピペ**で成立させる（フレンドコード方式）：

```
1. 自分の成体ペットの「おみあいコード」を発行
   例: INU-7K2M-QX9A-3F （species/品種/性格/色などをエンコード＋チェックサム）
2. LINEやSNSで友だちに送る（共有シートを利用。アプリは通信しない）
3. 相手のコードを自分のアプリに入力 → おみあい成立
4. 自分の成体は巣立ち（図鑑に登録）、ふたりの特徴を継いだおくるみが届く
```

- コードは20文字程度のbase32。**端末外への自動送信はゼロ**（手動コピペ）→ データセーフティ申告は「収集なし」のまま
- チェックサムで打ち間違い検出。改ざんはカジュアル対策のみ（ローカルゲームなので実害なし）
- ひとりでも遊べる救済: アプリ内に「ご近所さんのコード」を数枚同梱（NPCおみあい）**[v1.2検討]**

## 3. 遺伝システム（ランダム継承の設計）

子は「ミックス」という特別な存在。**遺伝子＝artパラメータ＋性格**を親からランダム継承する：

| 遺伝子 | 継承ルール |
|---|---|
| 体色 color / 副色 color2 | 各50%でどちらかの親から。**8%で突然変異**（全品種の色プールからランダム） |
| 耳 ear / 模様 pattern / ふわふわ fluffy | 各50%で親から。突然変異5% |
| 目の色 eye | 50%で親から。**まれに金銀の希少色（2%）** |
| 性格 nature | 50%でどちらかの親。**5%で「ハイブリッドな新性格」**（例:「じゆうじん」= ミックス限定性格） |
| 名前のヒント | 親2匹の名前/品種を記録（「シバ と コーギー の子」と図鑑に残る） |

- **同品種同士（柴犬×柴犬）**: 純血の柴犬が生まれ、図鑑カウント+1。レア運ボーナス大
- **雑種強勢ボーナス**: ミックスの子は成長速度+20%（生物学ネタの遊び。おさんぽが楽しくなる）
- 組み合わせ空間: 耳4×模様8×色∞×性格16 → **事実上無限のバリエーション**＝長期リテンションの核

## 4. 図鑑との関係 — 「ミックスのアルバム」

- 30種図鑑とは**別タブ「アルバム」**にミックスの子を記録（無限バリエーションのため図鑑コンプとは分離）
- 記録内容: 見た目スナップ・性格・親の記録・おみあい相手（コードの発行者名は任意入力）
- **「広告ゼロ・無課金で30種コンプ」の公約には触れない**設計（ミックスはコンプ対象外のおまけ宇宙）

## 5. マーケティング効果（なぜ「肝」になり得るか）

1. **バイラルループ**: コード交換は「友だちをアプリに誘う」行為そのもの。広告費ゼロ戦略における唯一の招待導線になる（「うちの柴とおみあいしない？」はダウンロードの誘い文句として自然）
2. **UGC**: 「うちの子とフォロワーさんの子のミックス」はSNS投稿の鉄板ネタ（#いぬねこ図鑑）
3. **差別化**: スマやめ・Forest・Focus Friendのいずれも対人メカニクスが弱い。デトックス系×ブリードは調査範囲で前例なし

## 6. 実装スケッチ（v1.1で2〜3日想定）

```
engine.js:
  encodeMate(state)            → おみあいコード文字列（成体のみ）
  decodeMate(code)             → {breedId, nature, art, name} | {error}
  breedWith(partner, now, rnd) → 自分の成体を巣立ち登録＋ミックスのおくるみ生成
  state.album[]                → ミックスの記録（セーブversion 3・マイグレーション）
art.js:
  petSVG は既に art パラメータ駆動 → ミックスの描画は追加実装ほぼゼロ（遺伝と相性が良い設計）
ui.js:
  成体時の巣立ちボタン横に「💞 おみあい」／コード表示・入力モーダル／アルバムタブ
```

- 単体テスト: コードのencode/decode往復・チェックサム・遺伝分布（rnd注入で決定論検証）・version3マイグレーション
- **v1.0には入れない理由**: 6/22凍結まで残り日数で実機検証を圧迫するため。Playの14日クローズドテスト期間（6/23〜7/上旬）がちょうどv1.1開発期間になる。テスター12人がそのままおみあい相手のテスト網になる

## 7. リスクと対策

| リスク | 対策 |
|---|---|
| コード手入力が面倒 | 共有シート連携＋クリップボード自動読み取り（同意トースト付き） |
| 1ペット制と矛盾（育成中はおみあい不可） | おみあい＝巣立ちの特別版として統合（成体のみ・現行ループを壊さない） |
| 図鑑コンプ公約との混線 | ミックスは別アルバム・コンプ対象外を明記 |
| 審査（UGC扱い） | コードは構造化データのみで自由文字列を含めない（名前はローカル表示限定）→ UGC非該当を維持 |
