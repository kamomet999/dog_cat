# ゲームデザイン仕様 — 生存システム（いのちと家出）

- 作成日: 2026-06-12 ／ 最終改訂: 2026-06-15（v7: **デスタイマーは前面稼働時のみ**＝スマホを使わない限り死なない） ／ 対象: v1.0（6/22凍結前）
- 発案者決定の積み上げ:
  1. 「ごはんがないと死ぬ・スマホを触っていない時間がないと死ぬ」
  2. 「なにがごはんになるかが大事」
  3. 「ごはん怠り→死 ／ さんぽ怠り→逃げる（家出）」
  4. 「ごはん＝スマホを触らない時間で餌が貰える・ストック制・自動給餌。さんぽ＝学習や運動など"いい時間"・リミット長め」
  5. （2026-06-13）「ごはん＝**スマホをロックして誘惑を断つ時間**。長くロックするほど餌が増える。ポケモンスリープのように寝る前/仕事中/学校で意図的にロックし、このアプリで解除する。電話と許可アプリ（Kindle等の自己研鑽）は例外」
  6. （2026-06-13）「さんぽ＝**自分で課題設定して取り組む時間**。勉強時間やダイエットを可視化。**継続日数・時間スコアのダッシュボード**が欲しい。**時間指定アラート**でペットから『さんぽしないの？』と促す」
  7. （2026-06-13）「**さんぽでも餌がもらえる設計にして良い**」
  8. （2026-06-13・技術判断）「本体ロック／他アプリ遮断／『電話だけ可』／アプリ許可リストはOS制約が大きい（iOS=Screen Time API＋特別entitlement承認、Android=UsageStats＋オーバーレイ）。**v1は両OSともオナーシステムのおすわり（離席＝背面化を検知）**で実現し、本格ロックは**v2**へ。Kindle等の自己研鑽は『さんぽ課題』として餌になるので実害は出ない」
  9. （2026-06-15・**重要な設計変更**）「**スマホをずっと使わないなら死なない**設計でなければならない。**デスタイマー（いのち・さんぽの減衰／死・家出）は、アプリが前面＝スマホが稼働しているときだけ進む**。アプリを閉じて離れている間は減らない（むしろ たまった在庫で自動給餌・回復する）。＝デジタルデトックスの思想と一致：スマホから離れるほど この子は安全に育ち、死ぬのは“スマホを使い続けて世話を怠ったとき”だけ」
     - 帰結: 「えさ＝アプリを閉じた時間」だけが餌になる（前面では受動の餌は貯まらない）。留守中の危険予告通知は廃止（`dangerForecast`は常に空＝責めない・途切れる予告をしない）。死/家出の2エンディング自体は維持（前面で世話を怠れば訪れる）。

## 1. コンセプト

> **この子は「あなたがスマホを置いた時間」を食べて生き、「あなたのいい時間」に寄り添って暮らす。**

ごはんとさんぽは「**断つ（受動）×励む（能動）**」で軸を分ける。被らない。

| | **ごはん（おすわり）** | **さんぽ（課題セッション）** |
|---|---|---|
| 本質 | スマホを**ロックして誘惑を断つ** | 自分で決めた課題に**取り組む** |
| 源泉 | ロックして触らなかった時間（**長いほど増量**） | 学習・読書・運動・ダイエットなど自分の課題の時間 |
| 餌 | 主獲得（ロック時間に比例） | **取り組んだぶんも餌がもらえる**（ボーナス） |
| 仕組み | ロックを開始しスマホを置く → 戻らずに満了で成立（早く戻ると中断・60秒猶予）／自動給餌（ストック制） | 課題セッション（**失敗なし**）でさんぽゲージ回復＋餌＋**ダッシュボード記録** |
| 付加価値 | — | **継続日数・累計時間・スコアの可視化**＋**時間指定リマインド** |
| 怠ると | **おほしさま（死）** — ただし**前面で**おなか0＋在庫切れを放置し続けたときだけ（離れている間は死なない・v7） | **家出（旅に出る）** — 同じく前面でゲージ0を長く放置したときだけ |
| 通知 | （枯渇予告は廃止＝責めない） | 指定時刻の自己設定リマインド「さんぽしないの？🐾」のみ |

## 2. ごはん（おすわり）＝ スマホを置く時間

```
獲得
  おすわり（主獲得・オナーシステム・Forest型）
    「おすわり」を開始 → スマホを置いて過ごす。選んだ時間 戻ってこなければロック成立＝餌。
    長いほど増量: 30分=2 / 60分=4 / 120分=8 / 180分=12（WALK_OPTIONS・walkFoodGain）。獲得上限は 24h ぶん。
  受動回復（アプリを閉じている時間）   : 0.1えさ/h（2.4/日・報酬と同じ24h上限）
  コイン購入（コイン=放置時間の蓄積）   : 20コイン/1えさ
  ストック上限21（約1週間ぶん）・初期6

判定（責めない）
  満了前にアプリへ戻る＝中断（早すぎる"解除"。連続記録だけリセット・責めない）。
  開始から60秒は猶予（誤タップ・着信の救済。WALK_GRACE）。
  ※実装は既存の walk（startWalk / checkWalk / _finishWalk・walkStats）を「おすわり」として用いる。

電話・許可アプリ（事前選択・v1はオナー）
  許可アプリ: ロック中に使ってよいアプリ（Kindle・Duolingo・電話 等）を せってい/ロック画面で事前選択（候補＋自由入力）。
  ロック画面に一覧を表示し、URLスキームがあれば直接起動するショートカットを置く（置いたまま離席＝ロック継続）。
  v1の限界: OSが「どのアプリを開いたか」を返さない（特にiOS）ため、許可アプリの自動判定や強制遮断はしない（オナー）。
    本モデルでは「アプリへ早く戻る」ことだけが中断条件なので、他アプリの使用自体は中断にならない。
    Kindle等の自己研鑽は「さんぽ課題」としても餌になるので、v1でも体験は成立する。
  v2: §8。許可リスト以外をOSレベルで実際に遮断＋使用時間を自動計測（iOS ScreenTime / Android UsageStats）。
    v1で選んだ許可リストはそのままv2の遮断対象に流用する。

消費（自動給餌）
  おなか<40 になったら自動で1えさ（おなか+60）。おなか減衰 -100/12h ≒ 1日3えさ
  「てであげる」と なかよし+4 のボーナス（自動にはない・絆の演出）

死（前面稼働中のみ進む・責めない）
  ストック切れ→おなか0→ いのち -100/24h ／ おなか>30で +100/48h 回復
  ※ この減衰は **アプリが前面（スマホ稼働中）のときだけ** 進む（v7・決定9）。
  完全放置（アプリを閉じて離れている）: いのちは減らない。むしろ在庫で自動給餌・回復＝安全。
  死が訪れるのは「アプリを開いたまま おなか0＋在庫切れを 放置し続けたとき」だけ。
  （責めないトーン・いのちハート・煽る予告はしない）
```

## 3. さんぽ（いい時間・課題セッション）＝ 励む時間

```
種類: どくしょ / えいご / うんどう / ジョグ / ダイエット / しゅうちゅう（自分の課題）
長さ: 15 / 30 / 60 分 → さんぽゲージ +35 / +50 / +80（上限100）
餌:   成功で餌ボーナス（取り組んだ時間に比例）。←【新】さんぽでも餌がもらえる
きせかえ: 成功ごとに確率(約50%)で『きせかえ』(ペットのアクセサリ全12種)をランダム入手。無料コレクション・入手時にアナウンス＋「着せる？」確認（自動装備しない・👕クローゼットでも着替え）。
場所:  全画面で散歩先(こうえん/かわ/まち/やま/うみ/よみち)を選ぶ＝景色。背景は生成画像 assets/places/<id>.jpg。歩く姿は <id>_walk スプライト。
性質: 失敗なし。開始したら他アプリ（Kindle・英語アプリ等）を使ってOK。
      途中でやめても「またこんど」（ペナルティなし）
ゲージ: -100/168h（1週間で空）。0が72時間つづくと家出 ＝ 計10日サボると旅に出る
        ※ この減衰も **前面（スマホ稼働中）のときだけ** 進む（v7・決定9）。離れている間は減らない＝放置で家出はしない
家出: 「あたらしい家族をさがしに行く」（死ではない・責めないトーン・図鑑非登録・runaways記録）
```

### 3.1 ダッシュボード（新規）

> 発案者: 「勉強時間やダイエットの取り組みを可視化したい。継続日数や時間のスコアがほしい」

- **継続日数（streak）**: その日にさんぽ成功した日が連続した数。途切れたら0から。
- **累計時間 / 今週の時間**、**種類別の内訳**（勉強/運動/ダイエット…）で可視化。
- **スコア**: 例 = 継続日数の重み + 累計時間。前向きな称号/レベルで表示（責めない原則。減点・赤字表現を煽りに使わない）。

### 3.2 時間指定リマインド（新規）

- ユーザーが好きな時刻（複数可）を設定 → 毎日その時刻に「さんぽしないの？🐾」をローカル通知（daily repeat）。
- 既存の「枯渇予測」通知（§4）とは別枠。オン/オフと時刻はせってい画面から。

## 4. 通知（責めない・良い知らせと お世話のお願いのみ）

> v7（決定9）で方針転換: 離れている間は いのちが減らない＝**留守中に「危険」は起きない**。
> よって「えさがなくなりそう」等の **枯渇予告通知は廃止**（`dangerForecast` は常に空）。
> 煽って戻させる（途切れる予告）のは DESIGN.md 原則2「責めない」に反するため採らない。

- 残す通知は **ユーザーが自分で設定した 時間指定リマインドのみ**（§3.2）。
  - 「さんぽ しないの？🐾 きょうの いいじかん、いっしょに」（daily repeat・オン/オフと時刻は設定画面）
- バックグラウンド移行時の自動危険予約（旧 `dangerForecast`→onPause）は行わない（空をスケジュール）。
- Androidは重要度最高のチャンネル／iOSは標準バナー（リマインド用）。

## 5. おわかれ／旅立ちの設計

- どちらも**責めないトーン**・赤/警告アイコン禁止・図鑑には登録されない（図鑑=最後まで育てた記録）
- おほしさま🌟「いっしょにすごしたじかんはきえないよ」／旅立ち🎒「どこかでげんきにしているよ」
- 記録: せってい画面に件数。**課金での復活・延命は永久になし**（DESIGN.md 変えない約束）
- 直後に無料で新しい子をおむかえ（立ち直りを妨げない）

## 8. v2ロードマップ（本格ロック・OS連携）

v1はオナーシステム（アプリ内ロック＋離席検知）。本体ロック/他アプリ遮断は審査・権限ハードルが高く、Mac無し・Capacitor＋クラウドビルド構成では開発/テストも重いため v2 研究テーマとする。

- **iOS**: Screen Time API（FamilyControls / ManagedSettings / DeviceActivity）。`com.apple.developer.family-controls` 特別entitlementの**Apple承認**＋Swiftの**カスタムCapacitorプラグイン自作**が必要。これで指定アプリの shield（遮断）と使用時間計測が可能になる。
- **Android**: `UsageStatsManager`（PACKAGE_USAGE_STATS 特別権限）で前面アプリを検知し、**許可リスト（Kindle等）**以外をオーバーレイ（SYSTEM_ALERT_WINDOW）/アクセシビリティで遮断。Play審査ポリシーに注意。
- これにより「電話だけ可」「許可アプリ以外を本当に遮断」「他アプリ使用時間の自動計測」を実現する。
- ジョグ連携（歩数/GPS）も同枠。
