# ゲームデザイン仕様 — 生存システム（いのち）

- 作成日: 2026-06-12 ／ 対象: v1.0（6/22凍結前）
- 発案者決定: 「ごはんがないと死ぬ設計は残したい。**スマホを触っていない時間がないと死ぬ設計にしたい**」
- 旧方針「死なない・やさしい放置」を**改訂**する（死の実在＝たまごっちの感情の源泉。ただし伝え方は責めないトーンを維持）

---

## 1. コンセプト

> **この子は、ごはんと「あなたがスマホを置いた時間」を食べて生きている。**

- 生存条件は2つ: **おなか（ごはん）** と **おさんぽ充足（デトックス）**
- どちらかが尽きると衰弱し、放置すれば**おわかれ（死）**が訪れる
- きげん・きれい・げんきは生存条件ではなく**快適条件**（成長速度に効く。現行どおり avgStatus が成長係数）

### たまごっちの「ごはん／あそぶ」への回答

| 軸 | たまごっち | 本作 |
|---|---|---|
| おなか | ごはんで回復。空腹は死へ | 同じ（ごはんアクション。継承） |
| きげん | あそぶで回復。不機嫌は病気へ | あそぶで回復。**死なないが成長が遅くなる**（快適条件） |
| 本作固有 | — | **おさんぽ充足**: スマホを置いた時間（おさんぽ成功）でのみ回復。尽きると死へ |

## 2. ステータス設計

```
生存条件（尽きると いのち が減る）
  おなか hunger    0-100  減衰 -100/8h    回復: ごはん
  おさんぽ充足 detox 0-100  減衰 -100/72h  回復: おさんぽ成功のみ（+30+分/3, 上限100）
                                          30分=+40 / 1h=+50 / 2h=+70

いのち health 0-100（生死ゲージ・ハート表示）
  消耗: hunger=0 のあいだ -100/24h ／ detox=0 のあいだ -100/24h（両方なら加算=最速12hで-100）
  回復: hunger>30 かつ detox>30 のとき +100/48h
  0 で おわかれ

快適条件（成長係数 avgStatus に寄与・現行どおり）
  きげん mood / きれい clean / げんき energy
```

### 放置タイムライン（完全放置時の目安）

```
 0h   開始（全部100）
 8h   おなか0 → 衰弱開始
 32h  いのち0…にはまだ（おさんぽ充足が残っていれば -100/24hのみ → 32hで死）
      ※おさんぽもずっとしていない場合: detoxは72hで0 → 以降は二重消耗
最短コース: ごはんもおさんぽも一切なし → 約32時間でおわかれ
おさんぽだけサボる: ごはんは与え続けても 72h+24h ≒ 4日でおわかれ
```

- **シミュレーション上限 72h／報酬上限 24h に分離**: コイン・なかよしは従来どおり24h分まで。
  ただし生存シミュレーションは72h分まで進む＝**3日放置すれば帰ってきたときに結果（おわかれ含む）と向き合う**。
  それ以上の放置は72h分で打ち切り（開いていない端末では計算できないため）
- **おくるみ（stage0）は死なない**（ねんね中の赤ちゃんは保護される。生存消耗はstage1以降）

## 3. おわかれ（死）の設計 — 責めないトーンで、重さは本物

- いのち0 → 「**○○は おほしさまに なりました**」モーダル（静かに。罪悪感の煽り文言・赤・警告アイコン禁止）
- **図鑑には登録されない**（図鑑は巣立ち＝最後まで育てた記録。ここがたまごっち的な重さ）
- 記録: せってい画面に「おほしさまになった子: n」
- その後「あたらしい子をおむかえ」（無料・即時。立ち直りを妨げない）
- 巻き戻し・課金復活は**なし**（命の重さの一線。§DESIGN.md 変えない約束に追加）

## 4. 危険の予告（通知・表示）

衰弱は必ず**事前に知らせる**（フェアネス。気づけない死は理不尽）:

1. **アプリ内**: いのちハート表示（5個・health/20）。health<50でペットがしょんぼり、<25で横たわる表現（v1.1）＋復帰レポートに「ぐあいがわるそう…」
2. **ローカル通知**（アプリがバックグラウンドへ移行する瞬間に予測スケジュール・サーバー不要）:
   - おなか0の予測時刻: 「おなか ぺこぺこだよ…」
   - おさんぽ充足0の予測時刻: 「そろそろ スマホをおいて おさんぽしたいな」
   - いのち50%割れ予測: 「○○の ぐあいが わるいみたい」
   - 通知方針の改訂: 「良い知らせのみ」→ **「良い知らせ＋いのちに関わるお願い」**（ペット主語・責めない文言は不変。再エンゲージ目的の通知は引き続き禁止）
